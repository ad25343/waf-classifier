Module 1
🤖 Classification
Classify work items against the WAF — one at a time interactively, or thousands at once via async bulk processing.
💬
Single-Story Classification
Type a story title and description into a chat interface. The AI reads your WAF definitions and ground truth examples, returns a category, color, Run/Change flag, and confidence — with reasoning you can challenge in follow-up messages.
🤖 LLM
📦
Bulk File Verification
Upload a CSV or Excel of stories. Auto-detects columns, maps them, then classifies every row using concurrent AI workers. Results stream back as a progress bar — review, approve, or dispute each row before saving.
🤖 LLM
🗂
Column Mapping Preview
Before bulk verify runs, the tool parses your file and suggests which column maps to which field (title, WAF category, Team of Teams, story points, etc.). You correct the mapping before AI kicks off — no tokens spent.
✓ No LLM
✅
Approve to Ground Truth
One-click promotion of any verified story into your ground truth library. Future AI calls use it as a calibration example, improving consistency over time as your dataset grows.
✓ No LLM
Module 2
📊 Analytics & History
Understand how your backlog is classified — across teams, time periods, and WAF categories — with flexible exports for stakeholders.
🏠
Dashboard Summary
At-a-glance totals: stories classified, approval rate, mismatch rate, top categories, color distribution, and daily trend — filterable by team, upload, or date range.
✓ No LLM
📅
Sprint & Monthly Rollups
Buckets history into 2-week sprints or calendar months. Shows category shifts, color changes, and confidence trends with period-over-period deltas so you can track WAF alignment over time.
✓ No LLM
🔍
Story Drilldown
Full-text searchable table of every classified story. Filter by team, category, color, match/mismatch, date. Expand any row to see AI reasoning, original tag, and full classification detail.
✓ No LLM
📝
AI Executive Narrative
Generates a concise, human-readable paragraph from your summary stats — describing WAF alignment, risk areas, and team performance. Ready to paste into a slide deck or status update.
🤖 LLM
📥
CSV / Excel Export
Download filtered history as a flat CSV or formatted multi-sheet Excel workbook (Summary, Monthly, and Raw Data tabs) for offline analysis or stakeholder sharing.
✓ No LLM
⬆️
Bulk Import
Upload a pre-classified CSV/Excel to populate the history database without running AI — preserving timestamps and original tags from previous tools or manual tagging exercises.
✓ No LLM
Module 3
🔀 File Merge
Join separate JIRA exports — Epics, Features, and Stories — into one clean, enriched file ready for classification or reporting.
🧩
Epic → Feature → Story Join
Takes up to three JIRA export files and joins them by name (case-insensitive). Extracts WAF color and category, strips Run/Change suffixes from epic names, and assigns PI Name from the Epic file down to each story row.
✓ No LLM
⚠️
Issue Detection
Flags four problem types during merge: orphaned stories (no matching feature), orphaned features (no matching epic), stories missing WAF data, and unrecognised WAF colors. You choose which rows to include before downloading.
✓ No LLM
🚀
Send to Classifier
One-click handoff from the merge result directly into the Bulk Verify flow — no intermediate download needed. The merged file arrives with column detection already run and fields pre-mapped.
✓ No LLM
Output Schema (18 columns)
PI Name Epic Id Epic Name Epic Desc Block WAF WAF Color WAF Category Run/Change Feature Id Feature Name Feature Desc Team of Teams Story Id Story Name Story Desc Story Points Assigned Teams
Module 4
👥 Teams
Understand how every Scrum Team and Team of Teams (ART) is aligned to the WAF — and how their work is distributed across epics.
📋
Team Summary
For every Scrum Team: total stories, unique epics touched, WAF category breakdown, dominant color, mismatch rate, and their Team of Teams (ART). Supports filtering by Team of Teams for ART-level views.
✓ No LLM
🔎
Team Drilldown
Drill into a single team to see their work organised as Epic → Feature → Story trees, with per-epic category consistency, dominant classifications, and health indicators.
✓ No LLM
🗺
Teams by Epic
Flip the view — pick an epic and see every team contributing to it, their story counts, and category distribution. Immediately surfaces cross-team alignment (or divergence) on a single initiative.
✓ No LLM
📜
Epics List
A master list of all epics with story counts, team assignments, and dominant WAF category — used as the navigation starting point for the By Epic tab and cross-team epic analysis.
✓ No LLM
Hierarchy:
Team of Teams (ART) → Scrum Teams → Stories
Module 5
🏆 Story Quality
Score every story against a 9-criterion Definition of Ready rubric — then chat with AI to refine or fully rewrite stories that don't pass.
🎯
Batch DoR Scoring
Scores every story against 9 Definition of Ready criteria: Narrative Format, Source Data, Business Rules, Output Artifact, Acceptance Criteria, Data Quality Checks, Traceability, Story Points, Dependencies. Each gets pass/fail + reason.
🤖 LLM
💬
Story Improvement Chat
Open a chat against a specific story. The AI suggests targeted improvements for whichever criteria are failing — collaborative, iterative refinement without rewriting the whole story from scratch.
🤖 LLM
✍️
AI Story Rewrite
One click to have the AI produce a fully rewritten version of the story addressing all failing criteria, in the correct narrative format, ready to copy back into JIRA.
🤖 LLM
📊
Quality Results & History
Browse scoring results by upload, team, or scoring run. See aggregate Ready / Needs Work / Not Ready counts, per-criterion pass rates, and score trends across multiple runs.
✓ No LLM
📤
Quality Export
Download scoring results as CSV — one row per story with pass/fail columns for all 9 criteria, overall score percentage, and team/epic metadata for downstream reporting.
✓ No LLM
Score Bands
✅ Ready ≥ 89%
⚠️ Needs Work 56–88%
❌ Not Ready < 56%
Modules 6 & 7
🏗 Epic Lineage & ⚡ Disputes
Epic Lineage
❤️
Epic Health Scoring
Each epic gets a 0–100 health score: color consistency (40%), category focus (30%), distinct color count (20%), mismatch rate (10%). Epics spanning 3+ colors are flagged as "mixed".
✓ No LLM
🌳
Hierarchy Tree
Full Epic → Feature → Story tree showing which teams deliver each part, story counts per node, and dominant classifications at every level of the hierarchy.
✓ No LLM
🔗
Bulk Epic Assignment
Assign or re-assign Epic and Feature fields across multiple stories at once — essential for cleaning up imported data where those fields were blank or mismatched.
✓ No LLM
Disputes
🚩
Raise a Dispute
Flag any AI classification you disagree with. Submit your suggested category and a comment. The original AI reasoning is preserved alongside your correction for reviewer context.
✓ No LLM
📋
Dispute Queue
Filtered list of all disputes by status: Pending, Accepted, Dismissed, or Flagged for WAF Review. Shows who raised it, AI answer vs. suggested answer, and reviewer notes.
✓ No LLM
⚖️
Resolve a Dispute
Three paths: Dismiss (AI was right), Accept (save correction, optionally promote to ground truth), or Flag for WAF Review (the WAF definition itself may need updating).
✓ No LLM
Module 8
⚙️ Settings & Version Control
Manage the AI's knowledge base — WAF definitions and ground truth — with full version history, baselines, and system configuration.
📖
WAF Definitions Management
Upload a new WAF definitions file (CSV, Excel, JSON, or plain text) at any time. The tool parses categories, colors, and descriptions automatically. Edit definitions inline from the Settings page without re-uploading.
✓ No LLM
🎓
Ground Truth Management
Upload, add, edit, or delete ground truth examples — the calibration examples sent to the AI in every prompt. Quality of ground truth directly drives classification consistency across teams.
✓ No LLM
🏷
Named Versions
Save the current WAF definitions or ground truth as a named, timestamped version (e.g. "Q3 Baseline"). Switch between versions freely — bulk verify jobs record which version was active when they ran, ensuring reproducibility.
✓ No LLM
📸
Baseline Snapshots
Save a paired snapshot of both WAF + ground truth together as a named baseline. Restore in one click — ideal for audit trails, rolling back after an experiment, or matching a previous quarter's configuration.
✓ No LLM
System Config — No Code Needed
Batch size — stories per AI call
Workers — concurrent AI threads
Rate limit — calls per minute
Gateway — Anthropic / Bedrock / PortKey / Apigee
